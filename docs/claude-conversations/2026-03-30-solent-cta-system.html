Implementation CSS

Complete button system using CSS custom properties.

/* ═══════════════════════════════════════ CTA Button System ═══════════════════════════════════════ */ /* ── Base button ── */ .cta { display: inline-flex; align-items: center; gap: 0.5em; font-family: 'Atkinson Hyperlegible Next', sans-serif; font-size: 1rem; font-weight: 700; padding: 0.75em 1.5em; border: 3px solid transparent; border-radius: 4px; text-decoration: none; cursor: pointer; transition: all 0.2s ease; line-height: 1.2; } /* ── Caret (right-facing triangle) ── */ .cta-caret { display: inline-block; width: 0; height: 0; border-top: 5px solid transparent; border-bottom: 5px solid transparent; border-left: 6px solid currentColor; margin-left: 0.2em; transition: transform 0.2s ease; } .cta:hover .cta-caret { transform: translateX(3px); } /* ── Primary: solent blue fill, white text ── */ .cta-primary { background: var(--solent-blue); border-color: var(--solent-blue); color: white; } .cta-primary:hover { background: var(--solent-blue-light); border-color: var(--solent-blue-light); } /* ── Secondary: ghost with solent blue border ── */ .cta-secondary { background: transparent; border-color: var(--solent-blue); color: var(--solent-blue); } .cta-secondary:hover { background: var(--solent-blue); color: white; } /* ── On gradient: white border + solent fill ── */ .cta-on-gradient { background: var(--solent-blue); border-color: white; color: white; } .cta-on-gradient:hover { background: var(--solent-blue-light); } /* ── On gradient ghost: white border ── */ .cta-on-gradient-ghost { background: transparent; border-color: white; color: white; text-shadow: 0 1px 3px rgba(0,0,0,0.15); } .cta-on-gradient-ghost:hover { background: rgba(255,255,255,0.2); } /* ── On dark bg: white fill ── */ .cta-on-dark { background: white; border-color: white; color: var(--solent-blue); } .cta-on-dark:hover { background: var(--warm-grey); border-color: var(--warm-grey); } /* ── On dark bg ghost: white border ── */ .cta-on-dark-ghost { background: transparent; border-color: rgba(255,255,255,0.6); color: white; } .cta-on-dark-ghost:hover { background: rgba(255,255,255,0.1); border-color: white; }
Recommendations:

• Border radius: 4px. Friendly without being casual.
• Border width: 3px throughout. Creates a confident, visible edge without feeling heavy.
• Primary action: Solent blue fill everywhere except on solent blue backgrounds (where it inverts to white fill).
• Secondary action: Ghost (border only) in the same colour as the primary — fill on hover creates a satisfying interaction.
• On gradient: Always include the white border. Without it, the button disappears into the magenta section of the gradient.
• Magenta CTAs: Available in the system but use sparingly. If magenta is reserved for link underlines and menu selected state, using it for CTAs too could dilute its meaning. Solent blue as the primary CTA colour keeps it distinct from the magenta "interactive text" signal.
• Caret: The CSS triangle is cleaner than the chevron character, which can render differently across fonts. The subtle rightward nudge on hover gives a satisfying sense of forward motion.
• Override body link styles: CTAs should NOT inherit the magenta underline from .layout-content a. The text-decoration: none in the base .cta class handles this, but check specificity.
